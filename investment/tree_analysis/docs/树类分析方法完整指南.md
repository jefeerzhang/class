## 概述

### 什么是树类分析方法？

树类分析方法是一类基于树形结构进行决策和预测的机器学习算法。它们通过递归地将特征空间划分为多个区域，每个区域对应一个预测值或类别。这类方法因其**直观易懂、可解释性强**而广泛应用于各种领域，特别是金融、医疗等对可解释性要求较高的场景。

### 树类方法的发展历程

| 年代 | 方法 | 主要贡献 |
|------|------|----------|
| 1963 | AID (Automatic Interaction Detection) | 最早的回归树算法 |
| 1984 | CART (Classification and Regression Trees) | Breiman等人提出经典决策树框架 |
| 1993 | C4.5 | Quinlan改进ID3，处理连续值和缺失值 |
| 1995 | AdaBoost | Freund和Schapire提出boosting思想 |
| 2001 | 随机森林 | Breiman提出集成多个决策树的方法 |
| 2014 | XGBoost | 陈天奇优化梯度提升算法 |
| 2016 | LightGBM | 微软提出高效梯度提升框架 |

### 树类方法的优势

1. **可解释性强**：决策过程可视化，易于理解
2. **数据预处理简单**：无需特征缩放，能处理混合类型数据
3. **自动特征选择**：通过分裂准则自动选择重要特征
4. **非线性建模能力**：能捕捉复杂的非线性关系
5. **缺失值处理**：部分算法能直接处理缺失值
6. **计算效率适中**：训练和预测速度较快

## 决策树基本原理

### 树的基本结构

```
                    根节点 (Root Node)
                        |
              ┌─────────┴─────────┐
              |                   |
        内部节点 (Internal Node)   内部节点
              |                   |
        ┌─────┴─────┐         ┌─────┴─────┐
        |           |         |           |
      叶节点     叶节点      叶节点      叶节点
    (Leaf Node)                        
```

**核心概念：**

- **根节点**：包含所有样本的起始节点
- **内部节点**：进行特征分裂的节点
- **叶节点**：输出最终预测结果的终端节点
- **分支**：连接节点的路径，表示特征的分裂条件
- **深度**：从根节点到最远叶节点的路径长度

### 分裂准则

决策树通过选择最优特征和分裂点来划分数据，核心是定义"最优"的标准。

#### 1. 信息增益 (Information Gain) - 用于ID3/C4.5

**信息熵定义：**
$$H(D) = -\sum_{k=1}^{|K|} p_k \log_2 p_k$$

其中 $p_k$ 是第 $k$ 类样本所占比例，$|K|$ 是类别总数。

**信息增益：**
$$IG(D, A) = H(D) - \sum_{v=1}^{V} \frac{|D^v|}{|D|} H(D^v)$$

其中 $D^v$ 是特征 $A$ 取值为 $v$ 的样本子集。

**特点：**
- 信息增益越大，分裂后纯度提升越多
- 倾向于选择取值较多的特征（信息增益偏向问题）
- C4.5使用信息增益率改进此问题

#### 2. 基尼系数 (Gini Index) - 用于CART

**基尼系数定义：**
$$Gini(D) = 1 - \sum_{k=1}^{|K|} p_k^2$$

**基尼增益：**
$$Gini\_Gain(D, A) = Gini(D) - \sum_{v=1}^{V} \frac{|D^v|}{|D|} Gini(D^v)$$

**特点：**
- 计算速度快（无需对数运算）
- CART算法使用基尼系数
- 取值范围 $[0, 1-\frac{1}{K}]$，越小越纯

#### 3. 均方误差 (MSE) - 用于回归树

**节点纯度度量：**
$$MSE(D) = \frac{1}{|D|} \sum_{i \in D} (y_i - \bar{y})^2$$

其中 $\bar{y}$ 是节点内样本目标值的均值。

**分裂准则：**
选择使分裂后加权MSE最小的特征和分裂点：
$$\min_{j,s} \left[ \sum_{i \in D_L(j,s)} (y_i - \bar{y}_L)^2 + \sum_{i \in D_R(j,s)} (y_i - \bar{y}_R)^2 \right]$$

### 停止条件

决策树通过以下条件停止分裂：

1. **节点纯度达标**：节点内样本属于同一类别（分类）或方差足够小（回归）
2. **样本数量阈值**：节点样本数小于预设阈值
3. **深度限制**：达到最大深度限制
4. **信息增益阈值**：分裂带来的增益小于阈值
5. **特征耗尽**：没有可用特征进行分裂

### 剪枝策略

#### 预剪枝 (Pre-pruning)

在构建过程中限制树的生长：
- 最大深度限制 (`max_depth`)
- 节点最小样本数 (`min_samples_split`, `min_samples_leaf`)
- 分裂最小增益 (`min_impurity_decrease`)

#### 后剪枝 (Post-pruning)

构建完整树后进行简化：
- **代价复杂度剪枝 (CCP)**：$\min_{T \subset T_0} \{ R_\alpha(T) = R(T) + \alpha |T| \}$
  - $R(T)$：树的预测误差
  - $|T|$：叶节点数量
  - $\alpha$：复杂度参数

**剪枝对比：**
| 方法 | 优点 | 缺点 |
|------|------|------|
| 预剪枝 | 计算效率高，避免过拟合 | 可能欠拟合 |
| 后剪枝 | 欠拟合风险低 | 计算开销大 |

## 经典决策树算法详解

决策树的发展历程中，三个经典算法奠定了现代树类方法的基础：**ID3、C4.5和CART**。深入理解这三个算法的原理、区别与应用场景，是掌握树类方法的关键。

### 三大算法对比概览

| 特性 | ID3 | C4.5 | CART |
|------|-----|------|------|
| 提出时间 | 1986年 | 1993年 | 1984年 |
| 提出者 | Ross Quinlan | Ross Quinlan | Breiman等 |
| 分裂准则 | 信息增益 | 信息增益率 | 基尼系数 |
| 特征类型 | 仅类别特征 | 类别+连续特征 | 类别+连续特征 |
| 树结构 | 多叉树 | 多叉树 | 二叉树 |
| 缺失值处理 | 不支持 | 支持 | 支持（代理分裂） |
| 剪枝方法 | 无 | 错误率降低剪枝 | 代价复杂度剪枝 |
| 任务类型 | 仅分类 | 仅分类 | 分类+回归 |
| 计算复杂度 | 较高（对数运算） | 较高（对数运算） | 较低（无对数） |

### ID3算法

ID3（Iterative Dichotomiser 3）是Ross Quinlan在1986年提出的第一个经典决策树算法，奠定了信息论方法在决策树构建中的基础。

#### 核心思想

ID3的核心思想是**信息熵与信息增益**。每次选择能够提供最大信息增益的特征进行分裂，使分裂后节点的"不确定性"最大化降低。

#### 信息熵详解

**信息熵的直观理解：**

信息熵度量系统的"不确定性"或"混乱程度"。熵越高，系统越不确定；熵越低，系统越确定。

- **纯节点**（所有样本属于同一类）：熵 = 0
- **最混乱节点**（各类样本均匀分布）：熵达到最大值 $\log_2(K)$

**数学定义：**

$$H(D) = -\sum_{k=1}^{K} p_k \log_2 p_k$$

其中：
- $K$ 是类别数量
- $p_k = \frac{|C_k|}{|D|}$ 是第 $k$ 类样本所占比例
- 约定 $0 \log_2 0 = 0$

**二分类问题示例：**

假设某节点有100个样本，其中70个正类，30个负类：

$$H(D) = -\frac{70}{100} \log_2 \frac{70}{100} - \frac{30}{100} \log_2 \frac{30}{100} = 0.881$$

如果节点完全纯（100个样本全是正类）：

$$H(D) = -1 \cdot \log_2 1 = 0$$

#### 信息增益计算

**定义：**

$$IG(D, A) = H(D) - \sum_{v=1}^{V} \frac{|D^v|}{|D|} H(D^v)$$

其中：
- $D$：当前节点的样本集合
- $A$：待评估的特征
- $V$：特征 $A$ 的取值数量
- $D^v$：特征 $A$ 取值为 $v$ 的样本子集

**直观理解：** 信息增益 = 父节点熵 - 子节点熵的加权平均

**计算步骤：**

1. 计算当前节点熵 $H(D)$
2. 对特征的每个取值 $v$，计算子节点熵 $H(D^v)$
3. 计算加权平均子节点熵
4. 信息增益 = 父节点熵 - 加权平均子节点熵

#### ID3算法流程

```
算法：ID3决策树构建
输入：训练数据集 D，特征集 A
输出：决策树 T

1. 若 D 中所有样本属于同一类 C，则 T 为单节点树，类标记为 C，返回 T

2. 若 A 为空集，则 T 为单节点树，类标记为 D 中样本数最多的类，返回 T

3. 否则，计算 A 中每个特征对 D 的信息增益：
   a. 对每个特征 a ∈ A
   b. 对 a 的每个取值 v，划分 D 得到 D_v
   c. 计算信息增益 IG(D, a)

4. 选择信息增益最大的特征 a_max 作为当前分裂特征

5. 对 a_max 的每个取值 v_i：
   a. 构建子节点
   b. 令 D_i = {(x,y) ∈ D | a_max(x) = v_i}
   c. 若 D_i 为空，则该子节点标记为 D 中样本数最多的类
   d. 否则，以 D_i 和 A - {a_max} 为参数递归构建子树

6. 返回决策树 T
```

#### ID3的局限性

**1. 信息增益偏向问题**

信息增益倾向于选择取值较多的特征（高基数特征），即使这些特征可能并不真正重要。

**示例说明：**

假设有一个"身份证号"特征，每个样本的身份证号都唯一。按此特征分裂后，每个子节点只含一个样本，熵为0，信息增益达到最大。但这显然没有意义——新样本的身份证号几乎不会与训练集重复。

**2. 仅支持类别特征**

ID3无法直接处理连续型特征（如年龄、收入），需要预先离散化。

**3. 不支持缺失值**

当数据存在缺失值时，ID3无法有效处理。

**4. 无剪枝机制**

ID3倾向于构建过深的树，容易过拟合。

**5. 多叉树结构**

每次分裂产生多个分支，导致树结构复杂，且特征使用一次后不能再使用。

#### ID3适用场景

- 特征均为类别型且基数适中
- 数据完整无缺失值
- 需要快速原型验证

### C4.5算法

C4.5是Quinlan在1993年提出的ID3改进版，解决了ID3的主要局限性，成为决策树发展史上的里程碑。

#### C4.5的改进点

**改进1：信息增益率（解决偏向问题）**

**信息增益率定义：**

$$Gain\_Ratio(D, A) = \frac{IG(D, A)}{IV(A)}$$

其中 $IV(A)$ 是特征的固有值（Intrinsic Value）：

$$IV(A) = -\sum_{v=1}^{V} \frac{|D^v|}{|D|} \log_2 \frac{|D^v|}{|D|}$$

**固有值的含义：**

$IV(A)$ 度量特征取值分布的熵。取值越多且分布越均匀，$IV(A)$ 越大。

**修正原理：**

| 特征类型 | 信息增益 | 固有值 | 信息增益率 |
|----------|----------|--------|------------|
| 高基数特征 | 很高 | 很大 | 可能不高 |
| 低基数特征 | 中等 | 较小 | 可能更高 |

**启发式策略：**

C4.5并非直接选择增益率最大的特征，而是：

1. 找出信息增益高于平均值的特征
2. 在这些特征中选择增益率最高的

这样避免过度偏向低基数特征。

**改进2：连续特征处理**

C4.5采用**二分法**处理连续特征：

```
连续特征处理算法：
输入：连续特征 a，样本集 D
输出：最优分裂点

1. 对特征 a 的所有取值排序：v_1 < v_2 < ... < v_n

2. 计算候选分裂点集合：
   S = {(v_i + v_{i+1}) / 2 | i = 1, ..., n-1}

3. 对每个分裂点 s ∈ S：
   a. 将样本分为两部分：
      D_≤ = {(x,y) | a(x) ≤ s}
      D_> = {(x,y) | a(x) > s}
   b. 计算信息增益 IG(D, s)

4. 选择信息增益最大的分裂点作为最优分裂点
```

**改进3：缺失值处理**

C4.5采用概率分配策略处理缺失值：

**分裂时处理：**

假设特征 $A$ 有缺失值，计算信息增益时：
- 只使用特征 $A$ 非缺失的样本 $D_{known}$
- 信息增益按比例调整：$IG(D, A) = \frac{|D_{known}|}{|D|} \cdot IG(D_{known}, A)$

**样本分配时处理：**

样本在特征 $A$ 上有缺失值时，同时进入所有子节点，权重按子节点样本比例分配：

$$weight(D^v) = weight(D) \times \frac{|D^v|}{|D_{known}|}$$

**改进4：剪枝机制**

C4.5采用**错误率降低剪枝（REP）**：

将子树替换为叶节点，若替换后错误率增加不超过某阈值（考虑置信度），则剪枝。

**改进5：规则提取**

C4.5可以将决策树转化为等价的规则集，便于理解和部署。

#### C4.5算法流程

```
算法：C4.5决策树构建
输入：训练数据集 D，特征集 A（含类别和连续特征）
输出：决策树 T

1. 若 D 中所有样本属于同一类，创建叶节点，返回

2. 若 A 为空或样本数少于阈值，创建叶节点，标记为多数类，返回

3. 对每个特征 a ∈ A：
   a. 若 a 为类别特征：计算信息增益率
   b. 若 a 为连续特征：
      - 找最优分裂点
      - 计算该分裂的信息增益率

4. 选择增益率最高的特征/分裂点进行分裂

5. 处理缺失值样本（概率分配）

6. 对每个分支递归构建子树

7. 后剪枝优化

8. 返回决策树 T
```

#### C4.5的局限性

- 计算效率：信息增益率需要多次对数运算，速度较慢
- 内存开销：连续特征排序和分裂点计算消耗内存
- 多叉树仍可能导致过拟合
- 对大数据集效率不佳

### CART算法

CART（Classification and Regression Trees）由Breiman等人于1984年提出，是最具影响力的决策树算法之一，被scikit-learn等主流库采用。

#### CART的核心特点

**特点1：二叉树结构**

CART强制采用**二叉分裂**，每次分裂只产生两个子节点。这带来以下优势：

- 树结构简单，易于理解和维护
- 特征可多次使用（不同层级可用同一特征）
- 避免多叉树的数据碎片化问题

**二叉分裂实现：**

| 特征类型 | 分裂方式 |
|----------|----------|
| 类别特征 | 选择最优子集划分（如 {A,B} vs {C,D}） |
| 连续特征 | 选择最优分裂点（如 ≤50 vs >50） |

**特点2：基尼系数**

**基尼系数定义：**

$$Gini(D) = 1 - \sum_{k=1}^{K} p_k^2$$

**直观理解：**

基尼系数度量从数据集 $D$ 中随机抽取两个样本，它们类别不同的概率。

- 纯节点：$Gini = 0$
- 最混乱（均匀分布）：$Gini = 1 - \frac{1}{K}$

**基尼系数 vs 信息熵：**

| 方面 | 基尼系数 | 信息熵 |
|------|----------|--------|
| 计算速度 | 快（无对数） | 慢（需对数） |
| 数值范围 | [0, 1-1/K] | [0, log₂(K)] |
| 敏感性 | 对类别分布敏感 | 更敏感 |
| 实际效果 | 相近 | 相近 |

**特点3：支持回归**

CART通过最小化方差（MSE）构建回归树：

$$Gini\_Gain(D, s) = MSE(D) - \frac{|D_L|}{|D|} MSE(D_L) - \frac{|D_R|}{|D|} MSE(D_R)$$

叶节点的预测值为节点内样本目标值的均值。

#### CART分类树算法

**分裂准则：基尼指数最小化**

对于二分类问题，选择使分裂后基尼指数最小的特征和分裂点：

$$Gini\_index(D, A, s) = \frac{|D_L|}{|D|} Gini(D_L) + \frac{|D_R|}{|D|} Gini(D_R)$$

**类别特征的最佳分裂：**

对于有 $m$ 个取值的类别特征，CART需要找出最优的二分划分。

理论上需要枚举 $2^{m-1}-1$ 种划分方式。对于高基数特征，C4.5的多叉分裂可能更高效。

**连续特征的最佳分裂：**

与C4.5类似，排序后尝试所有候选分裂点，选择基尼指数最小的。

#### CART回归树算法

**分裂准则：最小化MSE**

$$\min_{j,s} \left[ \sum_{i \in D_L(j,s)} (y_i - \bar{y}_L)^2 + \sum_{i \in D_R(j,s)} (y_i - \bar{y}_R)^2 \right]$$

其中：
- $j$：特征索引
- $s$：分裂点
- $\bar{y}_L, \bar{y}_R$：左右子节点目标值均值

**叶节点预测：**

$$\hat{y}_{leaf} = \frac{1}{|D_{leaf}|} \sum_{i \in D_{leaf}} y_i$$

#### CART剪枝：代价复杂度剪枝（CCP）

**核心思想：**

平衡树的复杂度与预测误差，找到最优的复杂度参数 $\alpha$。

**代价复杂度度量：**

$$R_\alpha(T) = R(T) + \alpha \cdot |T|$$

其中：
- $R(T)$：树的误分类率（或误差）
- $|T|$：叶节点数量
- $\alpha$：复杂度参数（每增加一个叶节点的代价）

**剪枝过程：**

```
算法：代价复杂度剪枝
输入：完整树 T_0
输出：最优剪枝树 T_α

1. 递归地从 T_0 剪枝，生成一系列树 T_0, T_1, ..., T_k（T_k为根节点）

2. 对每个树 T_i，计算对应的 α_i：
   α_i = min{(R(T') - R(T)) / (|T| - |T'|)} 其中 T' 是 T 的子树

3. 使用交叉验证选择最优 α*

4. 返回 T_α*
```

**scikit-learn实现：**

```python

# 交叉验证选择最佳
```

#### CART的代理分裂（处理缺失值）

当最佳分裂特征存在缺失值时，CART使用**代理分裂**：

1. 找到与最佳分裂结果最相似的其他分裂（代理分裂）
2. 用代理分裂处理缺失值样本

**代理分裂的选择：**

代理分裂是与主分裂结果相关性最高的备选分裂，通过计算预测关联度确定。

#### CART算法流程

```
算法：CART决策树构建（分类）
输入：训练数据集 D，特征集 A
输出：二叉决策树 T

1. 创建根节点，包含所有样本

2. 若满足停止条件（纯度、样本数、深度），创建叶节点

3. 否则：
   a. 对每个特征和每种分裂方式计算基尼指数
   b. 选择基尼指数最小的特征和分裂点
   c. 将数据分为左右两部分

4. 递归对左右子集构建子树

5. 后剪枝：CCP剪枝优化

6. 返回决策树 T
```

### 三大算法深度对比

#### 分裂准则对比

| 准则 | 公式 | 适用算法 | 特点 |
|------|------|----------|------|
| 信息增益 | $H(D) - \sum p_v H(D^v)$ | ID3 | 偏向高基数特征 |
| 信息增益率 | $\frac{IG}{IV(A)}$ | C4.5 | 修正偏向问题 |
| 基尼系数 | $1 - \sum p_k^2$ | CART | 计算快，二叉分裂 |

#### 树结构对比

```
ID3/C4.5 多叉树示例：
          天气
    ┌─────┼─────┐
   晴天  雨天  雾天
    |     |     |
  叶节点 叶节点 叶节点

CART 二叉树示例：
          天气
        ┌───┴───┐
     晴天/雾天  雨天
        |       |
      叶节点   叶节点
```

#### 计算复杂度对比

假设有 $n$ 个样本，$p$ 个特征，特征 $j$ 有 $m_j$ 个取值：

| 算法 | 分裂计算复杂度 | 总复杂度 |
|------|----------------|----------|
| ID3 | $O(n \cdot m_j)$ per feature | $O(n \cdot \sum m_j)$ |
| C4.5 | $O(n \log n)$ for continuous | $O(n \cdot p \log n)$ |
| CART | $O(n \cdot m_j)$ for categorical | $O(n \cdot p)$ |

#### 实际应用选择指南

| 场景 | 推荐算法 | 原因 |
|------|----------|------|
| 纯类别特征、基数适中 | ID3/C4.5 | 信息论方法直观 |
| 含连续特征 | C4.5/CART | 支持连续特征处理 |
| 需要回归任务 | CART | 唯一支持回归的经典算法 |
| 追求计算效率 | CART | 基尼系数计算快 |
| 数据有缺失值 | C4.5/CART | 支持缺失值处理 |
| 需要规则提取 | C4.5 | 内置规则生成 |
| 使用scikit-learn | CART | sklearn默认实现 |

#### Python实现对比

```python
# ID3实现（需自定义，sklearn不直接支持）
from collections import Counter
import math

def entropy(y):
    """计算信息熵"""
    counts = Counter(y)
    probs = [count / len(y) for count in counts.values()]
    return -sum(p * math.log2(p) for p in probs if p > 0)

def information_gain(X, y, feature_idx):
    """计算信息增益"""
    total_entropy = entropy(y)
    feature_values = set(X[:, feature_idx])
    
    weighted_entropy = 0
    for value in feature_values:
        subset_y = y[X[:, feature_idx] == value]
        weighted_entropy += len(subset_y) / len(y) * entropy(subset_y)
    
    return total_entropy - weighted_entropy

# C4.5：使用信息增益率（sklearn不直接支持）
def intrinsic_value(X, feature_idx):
    """计算固有值"""
    counts = Counter(X[:, feature_idx])
    probs = [count / len(X) for count in counts.values()]
    return -sum(p * math.log2(p) for p in probs if p > 0)

def gain_ratio(X, y, feature_idx):
    """信息增益率"""
    ig = information_gain(X, y, feature_idx)
    iv = intrinsic_value(X, feature_idx)
    return ig / iv if iv > 0 else 0

# CART：sklearn直接支持
from sklearn.tree import DecisionTreeClassifier

# CART分类树（基尼系数）
cart_clf = DecisionTreeClassifier(
    criterion='gini',      # CART默认使用基尼系数
    max_depth=5,
    random_state=42
)

# 使用信息熵（模拟ID3/C4.5的分裂准则，但仍是二叉树）
id3_like = DecisionTreeClassifier(
    criterion='entropy',   # 使用信息熵而非基尼系数
    max_depth=5,
    random_state=42
)
```

**注意：** scikit-learn的 `DecisionTreeClassifier` 即使使用 `criterion='entropy'`，仍然是CART的二叉树结构，并非真正的ID3/C4.5多叉树。如果需要真正的ID3/C4.5，可以使用专门的库如 `决策树` 或自定义实现。

现代树类方法（随机森林、GBDT等）大多基于CART构建：

| 现代方法 | 基础树类型 | 原因 |
|----------|------------|------|

3. **通用性**：同时支持分类和回归
4. **剪枝成熟**：CCP剪枝理论基础扎实

## 分类决策树

### 原理详解

分类决策树用于预测离散类别标签。每个叶节点输出一个类别预测。

**预测规则：**
$$\hat{y} = \arg\max_k p_k(node)$$

即选择该节点中占比最大的类别作为预测结果。

### CART分类树算法流程

```
算法：CART分类树构建
输入：训练数据集 D, 停止条件参数
输出：决策树 T

1. 创建根节点，包含所有样本
2. 如果满足停止条件，创建叶节点，返回
3. 对每个特征和每个可能分裂点：
   a. 计算分裂后的加权基尼系数
   b. 记录最优特征和分裂点
4. 按最优特征和分裂点划分数据集
5. 对左右子集递归调用步骤2-4
6. 返回决策树
```

### 多类别处理

- **二叉分裂**：CART使用二叉树，每次分裂产生两个子节点
- **多路分裂**：ID3/C4.5可产生多路分裂，每个特征值一个分支

### 类别不平衡处理

```python
# 方法1：类别权重
class_weight = {0: 1, 1: 10}  # 少数类赋予更大权重

# 方法2：样本权重

# 方法3：过采样/欠采样
from imblearn.over_sampling import SMOTE

```

### scikit-learn实现

```python

    criterion='gini',          # 分裂准则：'gini' 或 'entropy'

    min_samples_leaf=5,        # 叶节点最小样本数
```

## 回归决策树

### 原理详解

回归决策树用于预测连续值。每个叶节点输出一个实数值预测。

**预测规则：**
$$\hat{y} = \frac{1}{|D_{leaf}|} \sum_{i \in D_{leaf}} y_i$$

即叶节点内所有样本目标值的均值。

### 分裂准则详解

回归树使用**方差减少量**作为分裂准则：

$$\Delta Var = Var(parent) - \sum_{child} \frac{n_{child}}{n_{parent}} Var(child)$$

选择使方差减少最大的特征和分裂点。

### MSE vs MAE

| 指标 | 公式 | 特点 |
|------|------|------|
| MSE | $\frac{1}{n}\sum(y_i - \hat{y})^2$ | 对异常值敏感，计算方便 |
| MAE | $\frac{1}{n}\sum|y_i - \hat{y}|$ | 对异常值鲁棒，计算较慢 |

```python
# 使用MSE
reg_mse = DecisionTreeRegressor(criterion='squared_error')

# 使用MAE
reg_mae = DecisionTreeRegressor(criterion='absolute_error')
```

### 线性回归树

部分实现支持在叶节点拟合线性模型而非常数：

$$\hat{y}(x) = \beta_0 + \beta_1 x_1 + ... + \beta_p x_p$$

可以减少树的深度，提高预测精度。

### scikit-learn实现

```python
from sklearn.tree import DecisionTreeRegressor

reg = DecisionTreeRegressor(
    criterion='squared_error',  # 'squared_error', 'absolute_error', 'poisson'
    max_depth=5,
    min_samples_split=10,
    min_samples_leaf=5,
    max_features='sqrt',
    random_state=42
)

reg.fit(X_train, y_train)

# 预测
y_pred = reg.predict(X_test)
```

### 回归树 vs 线性回归对比

| 方面 | 回归树 | 线性回归 |
|------|--------|----------|
| 非线性能力 | 强 | 弱（需特征工程） |
| 外推能力 | 差（预测值在训练值范围内） | 好 |
| 可解释性 | 规则清晰 | 系数含义明确 |
| 对异常值 | 相对鲁棒（需设置min_samples_leaf） | 敏感 |
| 特征缩放 | 不需要 | 必需 |

## 随机森林

### 原理详解

随机森林通过**Bagging**（Bootstrap Aggregating）和**随机特征选择**构建多棵决策树，综合它们的预测结果。

**核心思想：**
- **Bagging**：有放回抽样构建多个训练集
- **随机特征**：每次分裂随机选择特征子集
- **集成预测**：平均（回归）或投票（分类）

### 算法流程

```
算法：随机森林训练
输入：训练数据 D, 树的数量 B, 特征子集大小 m
输出：随机森林模型

For b = 1 to B:
    1. 从 D 中有放回抽取 n 个样本，得到自助样本 D_b
    2. 构建决策树 T_b：
       对每个节点：
       a. 从全部 p 个特征中随机选择 m 个特征
       b. 在这 m 个特征中选择最优分裂
       c. 分裂节点
    3. 得到树 T_b

预测：
- 回归：ŷ = (1/B) Σ T_b(x)
- 分类：ŷ = mode(T_1(x), ..., T_B(x))
```

### 超参数详解

```python
from sklearn.ensemble import RandomForestClassifier, RandomForestRegressor

rf = RandomForestClassifier(
    n_estimators=100,          # 树的数量
    max_depth=None,            # 每棵树的最大深度
    min_samples_split=2,       # 内部节点分裂最小样本数
    min_samples_leaf=1,        # 叶节点最小样本数
    max_features='sqrt',       # 每次分裂考虑的特征数
                                # 'sqrt': sqrt(n_features) - 分类推荐
                                # 'log2': log2(n_features)
                                # int/float: 指定数量或比例
    max_samples=None,          # 每棵树使用的样本数
    bootstrap=True,            # 是否使用自助采样
    oob_score=True,            # 是否计算袋外得分
    n_jobs=-1,                 # 并行计算
    random_state=42,
    class_weight='balanced'
)
```

### 关键超参数选择指南

| 参数 | 推荐范围 | 选择建议 |
|------|----------|----------|
| `n_estimators` | 100-500 | 越多越好，但有边际效应递减 |
| `max_depth` | None或10-30 | 不限制可能导致过拟合 |
| `max_features` | sqrt(n_features) | 分类推荐sqrt，回归推荐n_features/3 |
| `min_samples_leaf` | 1-5 | 增大可提高泛化能力 |

### 袋外误差 (Out-of-Bag Error)

**原理：** 每棵树约使用63.2%的样本训练，剩余36.8%的样本可用于验证。

$$OOB\_Error = \frac{1}{n} \sum_{i=1}^n L(y_i, \hat{y}_i^{OOB})$$

```python
rf = RandomForestClassifier(oob_score=True, n_estimators=100)
rf.fit(X_train, y_train)
print(f"OOB准确率: {rf.oob_score_:.4f}")
```

随机森林提供两种特征重要性度量：

#### 1. 平均不纯度减少 (MDI)

**对比：**

| 置换重要性 | 更可靠，可直接在测试集计算 | 计算慢，需要独立测试集 |

### 随机森林的优势

1. **高准确率**：通常优于单棵决策树
2. **抗过拟合**：多树集成降低方差
3. **并行化**：每棵树独立训练，易于并行
4. **特征重要性**：提供内置的特征重要性评估
5. **无需验证集**：可使用OOB误差评估

## 其他集成方法

### 梯度提升树 (Gradient Boosting)

**原理：** 顺序构建决策树，每棵树拟合前一棵树的残差。

$$F_m(x) = F_{m-1}(x) + \eta \cdot h_m(x)$$

其中 $h_m(x)$ 是第 $m$ 棵树对残差的拟合，$\eta$ 是学习率。

```python
from sklearn.ensemble import GradientBoostingClassifier, GradientBoostingRegressor

gb = GradientBoostingClassifier(
    n_estimators=100,          # 树的数量

    min_samples_split=10,
    subsample=0.8,             # 每棵树使用的样本比例
    random_state=42
)
```

## 模型比较与选择

### 单棵决策树

**优点：**
- ✅ 极强的可解释性，决策规则清晰
- ✅ 不需要特征缩放
- ✅ 能处理混合类型数据
- ✅ 训练和预测速度快
- ✅ 内存占用小

**缺点：**
- ❌ 容易过拟合
- ❌ 不稳定（数据微小变化导致树结构剧变）
- ❌ 决策边界为轴平行，无法处理复杂边界
- ❌ 外推能力差

**适用场景：**
- 需要高度可解释性的场景（如医疗诊断、信用审批）
- 数据量较小
- 特征交互关系较简单
- 需要快速部署的原型系统

### 随机森林

**优点：**
- ✅ 高准确率
- ✅ 抗过拟合能力强
- ✅ 无需特征缩放
- ✅ 提供特征重要性评估
- ✅ 可并行训练
- ✅ 能处理高维数据

**缺点：**
- ❌ 可解释性降低
- ❌ 模型体积大
- ❌ 预测速度比单棵树慢
- ❌ 对稀有特征不敏感

**适用场景：**
- 需要高准确率的预测任务
- 中等规模数据集
- 需要特征重要性分析
- 对可解释性有一定要求

### 梯度提升树 (GBDT/XGBoost/LightGBM)

**优点：**
- ✅ 通常达到最高精度
- ✅ 自动特征选择
- ✅ 能处理缺失值（XGBoost/LightGBM）
- ✅ 丰富的超参数可调

**缺点：**
- ❌ 容易过拟合（需调参）
- ❌ 训练时间较长（顺序训练）
- ❌ 超参数多，调参复杂
- ❌ 对异常值敏感

**适用场景：**
- 追求最高精度的竞赛/项目
- 数据量充足
- 结构化数据
- 有充足时间调参

### 选择决策流程图

```
开始
```

```python
def woe_encoding(df, feature, target):
    """
    对类别特征进行WOE编码
    """
    grouped = df.groupby(feature)[target].agg(['sum', 'count'])
    grouped['non_event'] = grouped['count'] - grouped['sum']

        (grouped['sum'] / grouped['sum'].sum()) / 
        (grouped['non_event'] / grouped['non_event'].sum())
```

### 投资组合优化

```python
import numpy as np
import pandas as pd
from sklearn.ensemble import RandomForestRegressor

# 预测各资产的预期收益
def predict_asset_returns(models, features):
    """
    预测多个资产的预期收益
    """
    predictions = {}
    for asset, model in models.items():
        predictions[asset] = model.predict(features)
    return pd.DataFrame(predictions)

# 构建投资组合
def optimize_portfolio(expected_returns, cov_matrix, risk_aversion=1.0):
    """
    均值-方差优化
    """
    from scipy.optimize import minimize
    
    n_assets = len(expected_returns)
    
    def objective(weights):
        portfolio_return = np.dot(weights, expected_returns)
        portfolio_risk = np.sqrt(np.dot(weights.T, np.dot(cov_matrix, weights)))
        # 最大化效用 = 收益 - 风险厌恶系数 * 风险
        return -(portfolio_return - risk_aversion * portfolio_risk)
    
    constraints = [
        {'type': 'eq', 'fun': lambda w: np.sum(w) - 1}  # 权重和为1
    ]
    bounds = [(0, 1) for _ in range(n_assets)]  # 非负权重
    
    initial_weights = np.ones(n_assets) / n_assets
    
    result = minimize(objective, initial_weights, 
                     method='SLSQP', 
                     bounds=bounds, 
                     constraints=constraints)
    
    return result.x
```

### 期权定价

```python
import numpy as np
from sklearn.tree import DecisionTreeRegressor
from sklearn.ensemble import RandomForestRegressor

# 模拟期权定价数据
def generate_option_data(n_samples=10000):
    np.random.seed(42)
    
    # 参数
    S = np.random.uniform(50, 150, n_samples)    # 标的价格
    K = np.random.uniform(40, 160, n_samples)     # 行权价
    T = np.random.uniform(0.1, 2, n_samples)      # 到期时间
    r = np.random.uniform(0.01, 0.1, n_samples)   # 无风险利率
    sigma = np.random.uniform(0.1, 0.5, n_samples) # 波动率
    
    # Black-Scholes公式计算理论价格
    from scipy.stats import norm
    d1 = (np.log(S/K) + (r + 0.5*sigma**2)*T) / (sigma*np.sqrt(T))
    d2 = d1 - sigma*np.sqrt(T)
    
    call_price = S * norm.cdf(d1) - K * np.exp(-r*T) * norm.cdf(d2)
    put_price = K * np.exp(-r*T) * norm.cdf(-d2) - S * norm.cdf(-d1)
    
    # 特征
    X = np.column_stack([S, K, T, r, sigma])
    y_call = call_price
    y_put = put_price
    
    return X, y_call, y_put

X, y_call, y_put = generate_option_data()

# 训练模型

    n_estimators=100,
    max_depth=15,
    random_state=42

model_call.fit(X, y_call)

# 评估
```

## 最佳实践与注意事项

### 数据预处理

1. **缺失值处理**
   - 树模型可以直接处理缺失值（如XGBoost、LightGBM）
   - 对于不支持的模型，使用中位数/众数填充
   - 可以将缺失作为一个特殊类别

2. **特征编码**
   ```python
   # 类别特征编码策略
   from sklearn.preprocessing import LabelEncoder, OrdinalEncoder, OneHotEncoder
   
   # 低基数特征：One-Hot编码
   low_cardinality = ['gender', 'marriage']
   
   # 高基数特征：目标编码或序数编码
   high_cardinality = ['city', 'occupation']
   ```

3. **特征缩放**
   - 树模型**不需要**特征缩放
   - 但如果与其他模型对比，建议保持一致性

### 超参数调优

```python
# 调参顺序建议
"""
1. n_estimators / num_boost_round: 先固定较大值
2. max_depth / num_leaves: 控制模型复杂度
3. min_samples_* / min_child_samples: 防止过拟合
4. learning_rate: 配合n_estimators调整
5. subsample / colsample_*: 增加多样性
6. 正则化参数: reg_alpha, reg_lambda
"""

# 自动调参工具
import optuna

def objective(trial):
    params = {
        'n_estimators': trial.suggest_int('n_estimators', 100, 500),
        'max_depth': trial.suggest_int('max_depth', 3, 12),
        'learning_rate': trial.suggest_float('learning_rate', 0.01, 0.3, log=True),
        'subsample': trial.suggest_float('subsample', 0.6, 1.0),
        'colsample_bytree': trial.suggest_float('colsample_bytree', 0.6, 1.0),
        'reg_alpha': trial.suggest_float('reg_alpha', 1e-8, 10.0, log=True),
        'reg_lambda': trial.suggest_float('reg_lambda', 1e-8, 10.0, log=True),
    }
    
    model = xgb.XGBClassifier(**params, random_state=42)
    model.fit(X_train, y_train)
    y_pred = model.predict_proba(X_test)[:, 1]
    return roc_auc_score(y_test, y_pred)

study = optuna.create_study(direction='maximize')
study.optimize(objective, n_trials=100)
print(f"最佳参数: {study.best_params}")
```

### 常见问题与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 过拟合 | 模型太复杂 | 增加正则化、减小树深度、增加min_samples |
| 欠拟合 | 模型太简单 | 增加树深度、减少正则化、增加n_estimators |
| 训练慢 | 数据量大/参数多 | 使用LightGBM、减少n_estimators、增加学习率 |
| 类别不平衡 | 少数类样本少 | 使用class_weight、过采样、SMOTE |
| 特征重要性偏向高基数特征 | MDI算法缺陷 | 使用置换重要性、SHAP值 |
