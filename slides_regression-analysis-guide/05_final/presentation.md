# 第一部分：回归分析基础

---

# 什么是回归分析？

**研究"因为A，所以B"关系的统计方法**

输入：各种影响因素（收入、年龄、信用记录）

输出：我们关心的结果（会不会违约、收益率多少）

**生活中的例子：**
- 房价预测：面积、位置、楼层如何影响价格
- 健康风险：年龄、血压、吸烟习惯如何影响心脏病风险
- 金融预测：客户画像如何影响违约概率

---

# 回归模型的分类

| 分类维度 | 类别 | 说明 |
|----------|------|------|
| **响应变量类型** | 定量响应变量 | 连续型结果（收益率、价格） |
| | 定性响应变量 | 分类型结果（违约与否、评级等级） |
| **响应变量数量** | 单变量回归 | 一个响应变量 |
| | 多变量回归 | 多个响应变量 |
| **模型形式** | 线性回归 | 线性关系假设 |
| | 非线性回归 | 含非线性变换 |

---

# 为什么需要定性响应变量回归？

**线性回归的问题：**

模型可能预测出"违约概率=120%"或"违约概率=-30%"

概率只能在0-100%之间！

**解决方案：**

使用特殊的"变形器"，把线性输出变成合理概率值

---

# 第二部分：定性响应变量回归模型

---

# 二分类Logistic回归

**Logistic函数（Sigmoid）**是一个"概率压缩器"：

$$P(违约) = \frac{1}{1 + e^{-z}}$$

- z很大时（100）→ 输出接近1
- z很小时（-100）→ 输出接近0
- z=0时 → 输出正好0.5

**发生比（Odds）：**
- 明天下雨概率80% → 发生比=4:1
- 发生比的Logit变换 → 可用线性模型建模

---

# Logistic回归系数解释

**系数就像"影响力评分"**

- 收入系数 = -0.5 → 收入越高，违约风险越低
- 负债率系数 = 0.3 → 负债越高，违约风险越高

**发生比解释：**
- 收入每增加1万元，不违约的发生比增加65%
- 负债率每增加10%，违约的发生比增加35%

---

# 多项Logistic回归

**适用场景：响应变量有多于两个的无序类别**

生活例子：
- 天气：晴天/多云/雨天/雪天
- 交通方式：开车/地铁/公交/骑车

金融应用：
- 投资风格：保守型/平衡型/激进型
- 市场状态：牛市/熊市/震荡市

**基准组对比：** 以最常见类别为基准，比较其他类别

---

# 有序Logistic回归

**适用场景：响应变量是有序类别**

生活例子：
- 电影评分：1星 < 2星 < 3星 < 4星 < 5星
- 满意度：非常不满意 < 不满意 < 一般 < 满意

金融应用：
- 信用评级：AAA > AA > A > BBB > BB > B > C
- 风险等级：低风险 > 中风险 > 高风险

**关键假设：** 每个因素对所有评级的影响是相同的

---

# Probit回归

**与Logistic的区别：**

| 方面 | Logistic | Probit |
|------|----------|--------|
| 变换函数 | 挤压函数 | 正态分布函数 |
| 尾部行为 | 变化较慢 | 变化较快 |
| 应用偏好 | 社会科学 | 金融经济学 |

**为什么金融领域喜欢Probit？**
- 理论基础：金融理论假设市场波动服从正态分布
- Merton模型：信用风险模型基于资产价值正态分布假设

---

# 第三部分：传统线性回归

---

# 线性回归模型形式

$$Y = \beta_0 + \beta_1 X_1 + \beta_2 X_2 + ... + \beta_p X_p + \epsilon$$

- Y：预测目标（房价）
- β₀：基础值
- β₁, β₂, ...：每个特征的影响力
- X₁, X₂, ...：特征（面积、楼层、位置）
- ε：误差项

**生活比喻：** 就像找"最佳裁判"，让所有数据点到裁判的距离最小

---

# OLS估计与推断

**最小二乘法原理：** 让所有点到直线的距离平方和最小

$$\hat{\beta} = (X^T X)^{-1} X^T Y$$

**性质：** BLUE（最优线性无偏估计）

**系数检验：**
- t检验：这个因素真的重要吗？
- p值：p<0.05说明影响显著
- F检验：模型整体是否显著

---

# 模型假设与检验

| 假设 | 检验方法 | 不满足会怎样 |
|------|----------|--------------|
| 线性性 | 残差图、RESET检验 | 预测不准 |
| 误差正态性 | Shapiro-Wilk、Q-Q图 | 推断不准 |
| 同方差性 | Breusch-Pagan、White检验 | 权重不等 |
| 无自相关 | Durbin-Watson | 时间序列出问题 |
| 无多重共线性 | VIF、条件数 | 系数不稳定 |

---

# 第四部分：正则化方法

---

# 为什么需要正则化？

**OLS的问题：**
1. **过拟合：** 模型记住无关细节（鞋码、早餐）
2. **多重共线性：** 身高体重高度相关，难以区分影响
3. **高维问题：** 100个特征，50个样本

**正则化就像"记忆限制"：**
- 只记住重要细节，忽略无关细节
- 给模型加上"限制"，防止过拟合

---

# 三种正则化方法对比

| 方法 | 效果 | 适用场景 |
|------|------|----------|
| **Ridge（L2）** | 系数缩小但不为0 | 多重共线性 |
| **Lasso（L1）** | 有些系数变为0（特征选择） | 高维稀疏 |
| **Elastic Net** | 结合两者优点 | 高维+共线性 |

**实际选择：**
- 特征数不多，共线性不严重 → Ridge
- 特征数很多，想做特征选择 → Lasso
- 特征数很多，又有共线性 → Elastic Net

---

# 第五部分：降维方法

---

# 为什么需要降维？

**问题场景：** 1000个特征预测学生成绩
- 每天早餐吃什么、鞋码多大、父母身高...

**降维就像"信息压缩"：**
- 原始照片：1000万像素，文件很大
- 压缩后：100万像素，关键信息还在

**降维策略：**
- 特征选择：从1000个特征选出最重要的10个
- 特征提取：把1000个特征合并成10个综合特征

---

# PCR vs PLS对比

| 方面 | PCR | PLS |
|------|-----|-----|
| 降维依据 | X的方差 | X与Y的协方差 |
| 利用Y信息 | 不利用 | 利用 |
| 预测能力 | 可能较差 | 通常更好 |
| 过拟合风险 | 较低 | 较高 |

**选择建议：**
- 预测准确性为主 → PLS
- 模型稳定性为主 → PCR
- 解释性需求 → PCR（主成分正交）

---

# 第六部分：模型选择与评估

---

# 定量响应变量评估指标

| 指标 | 含义 | 理想值 |
|------|------|--------|
| **MSE** | 均方误差 | 越小越好 |
| **RMSE** | 均方根误差 | 越小越好 |
| **MAE** | 平均绝对误差 | 越小越好 |
| **R²** | 解释方差比例 | 越大越好（接近1） |
| **MAPE** | 平均百分比误差 | 越小越好（<10%） |

---

# 定性响应变量评估指标

| 指标 | 含义 | 理想值 |
|------|------|--------|
| **准确率** | 分类正确率 | 越高越好 |
| **精确率** | 预测为"是"的真确率 | 越高越好 |
| **召回率** | 实际"是"被找出比例 | 越高越好 |
| **F1** | 精确率和召回率平衡 | 越高越好 |
| **AUC-ROC** | 区分能力 | 越高越好（>0.8） |
| **Log Loss** | 概率预测质量 | 越小越好 |

---

# 交叉验证

**核心思想：** 多次训练和验证，估计模型泛化能力

**K折交叉验证：**
1. 将数据划分为K个大小相等的子集
2. 每次使用K-1折训练，1折验证
3. 重复K次，计算平均验证误差

**常用K值：** K=5或K=10

**时间序列CV：** 金融数据需用时间序列交叉验证，避免未来信息泄露

---

# 子集选择方法

| 方法 | 原理 | 适用场景 |
|------|------|----------|
| **最佳子集** | 遍历所有组合 | p < 20 |
| **向前逐步** | 从空模型逐步添加 | p < 100 |
| **向后逐步** | 从全模型逐步移除 | p < 100 |
| **正则化** | 凸优化自动选择 | p > 100 |

**计算成本：** 最佳子集需要拟合2^p个模型

# 第七部分：金融应用案例

---

# 信用评分预测

**目标：** 预测借款人违约概率

**某银行案例发现：**
- 收入每增加1万元 → 违约概率下降15%
- 负债率超过60% → 违约概率是普通客户3倍
- 信用记录良好 → 违约概率仅为0.5%

**结果：** 优化贷款审批策略后，违约率下降30%

---

# 资产收益预测

**Fama-French三因子模型：**

$$R_i - R_f = \alpha + \beta_{MKT}(R_M - R_f) + \beta_{SMB}SMB + \beta_{HML}HML$$

- α：超额能力
- β_MKT：大盘敏感度
- β_SMB：小盘股效应
- β_HML：价值股效应

---

# 违约风险预测

**Merton结构化模型：**
- 企业价值 > 债务 → 股东赚钱
- 企业价值 < 债务 → 可能违约

**简化模型：**
$$PD = N\left(\frac{\log(D/V) - (\mu - \frac{1}{2}\sigma^2)T}{\sigma\sqrt{T}}\right)$$

**某银行发现：**
- 负债率超过80% → 违约概率是普通企业5倍
- 现金流为负 → 违约概率增加2倍

---

# 模型选择流程

**开始**
&nbsp;&nbsp;├── 响应变量类型？
&nbsp;&nbsp;|&nbsp;&nbsp;&nbsp;&nbsp;├── 定量 → 线性回归 → 多重共线性？→ 严重→Ridge/PCR/PLS
&nbsp;&nbsp;|&nbsp;&nbsp;&nbsp;&nbsp;|&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;轻微→OLS
&nbsp;&nbsp;|&nbsp;&nbsp;&nbsp;&nbsp;|&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;高维→Lasso/Elastic Net
&nbsp;&nbsp;|&nbsp;&nbsp;&nbsp;&nbsp;├── 定性（二分类）→ Logistic/Probit
&nbsp;&nbsp;|&nbsp;&nbsp;&nbsp;&nbsp;├── 定性（多分类）→ 多项Logistic
&nbsp;&nbsp;|&nbsp;&nbsp;&nbsp;&nbsp;└── 定性（有序）→ 有序Logistic
&nbsp;&nbsp;└── 评估与选择 → 预测为主→CV/AUC &nbsp;&nbsp;解释为主→AIC/BIC

---

# 总结：核心要点

1. **Logistic回归**是最常用的分类模型，可输出概率
2. **Ridge/Lasso**解决过拟合和多重共线性问题
3. **降维方法**处理高维特征，PLS考虑目标变量
4. **交叉验证**是评估模型泛化能力的关键
5. **正则化**与**子集选择**是两种特征选择途径

---

# 最佳实践建议

- **样本量：** 每个参数至少需要10-20个样本
- **类别不平衡：** 使用class_weight或SMOTE
- **特征标准化：** 正则化方法需要先标准化
- **模型监控：** 定期评估预测性能，监控模型漂移
- **可解释性：** Logistic回归用发生比比解释
