## DBSCAN：核心思想

**DBSCAN**（Density-Based Spatial Clustering of Applications with Noise）

> **簇是数据空间中密度较高的区域，被密度较低的区域分隔**

---

## DBSCAN：生活化理解

**城市人群分布**

- 商业区、住宅区、学校周围 → 人很密集，形成"簇"
- 道路、空地、公园 → 人比较稀疏，是"分隔带"
- 偏远郊区 → 人很少，可能是"噪声点"

---

## DBSCAN：关键概念

| 概念 | 解释 | 生活比喻 |
|------|------|----------|
| **ε (epsilon)** | 邻域半径 | 社交距离，关心周围10米内的人 |
| **MinPts** | 最小点数阈值 | 至少3人才算一群 |
| **核心点** | ε半径内有≥MinPts个点 | 社交达人，周围人很多 |
| **边界点** | 在核心点邻域内 | 边缘人物，依附于社交圈 |
| **噪声点** | 不属于任何群体 | 独行侠 |

---

## DBSCAN：工作过程

**就像组织社交活动**

1. **找核心人物**：先找一个社交达人（核心点）
2. **扩展圈子**：把这个人社交圈里的人都拉进来
3. **连锁反应**：如果圈子里还有其他社交达人，继续扩展
4. **边缘人物**：在圈子里但不是社交达人的，作为边界成员
5. **独行侠**：不属于任何圈子的，就是噪声点

---

## DBSCAN：参数选择

| 数据特征 | ε选择 | MinPts选择 |
|----------|-------|------------|
| 小数据集 | 较小（0.1-0.5） | 4-8 |
| 大数据集 | 较大（0.5-1.0） | 8-15 |
| 高维数据 | 需要更大 | 需要更大 |

**经验法则**：MinPts ≥ 数据维度 + 1

---

## DBSCAN：优缺点

**优点**

- 不需要预先指定簇数量
- 可以发现任意形状的簇
- 能识别噪声点
- 对异常值不敏感

**缺点**

- 对参数敏感
- 密度变化大时效果不好
- 高维数据效果差

---

## 层次聚类：核心思想

**像建立家族树一样，通过逐步合并来构建层次结构**

**班级合并比喻**

- 开始：每个学生自己是一个"组"
- 第一步：同桌的两人合并成一组
- 第二步：相邻的两组合并成更大的组
- 最后：整个班级合并成一个大组

---

## 层次聚类：两种方法

| 方法 | 方向 | 生活比喻 |
|------|------|----------|
| **凝聚型（自底向上）** | 从每个点开始，逐步合并 | 各自玩 → 找同桌玩 → 全班一起玩 |
| **分裂型（自顶向下）** | 从所有点开始，逐步分裂 | 全班一起玩 → 分成两半 → 各自玩 |

**大多数情况用凝聚型**

---

## 层次聚类：合并策略

| 策略 | 计算方法 | 生活比喻 |
|------|----------|----------|
| **单链接** | 最近点的距离 | 找朋友中最近的 |
| **全链接** | 最远点的距离 | 考虑最疏远的关系 |
| **平均链接** | 所有点对距离的平均值 | 综合评价 |
| **Ward方法** | 合并后簇内方差增加最小 | 保持圈子最紧密 |

---

## 层次聚类：实际例子

**客户细分**

| 客户 | 年龄 | 收入 | 消费频率 |
|------|------|------|----------|
| 客户1 | 25 | 5000 | 高 |
| 客户2 | 26 | 5500 | 高 |
| 客户3 | 45 | 20000 | 低 |
| 客户4 | 46 | 22000 | 低 |

**结果**：年轻高消费组 vs 中年高收入组

---

## 层次聚类：优缺点

**优点**

- 不需要预设簇数
- 树状图清晰展示层次关系
- 确定性算法，结果稳定

**缺点**

- 计算复杂度高：O(n³)
- 对噪声敏感
- 内存消耗大

---

## 算法对比

| 方面 | DBSCAN | 层次聚类 |
|------|--------|----------|
| 核心思想 | 基于密度 | 基于距离/层次 |
| 输出 | 平面划分 | 层次结构 |
| 需要预设簇数 | 否 | 否 |
| 发现任意形状 | 是 | 有限 |
| 处理噪声 | 好 | 较差 |
| 可扩展性 | 较好 | 差 |

---

## 如何选择算法

**选择 DBSCAN 的场景**

- 簇形状不规则
- 数据有噪声
- 不知道簇的数量

**选择层次聚类的场景**

- 需要层次结构
- 需要可视化
- 数据量较小

---

## Python 代码：DBSCAN

```python
from sklearn.cluster import DBSCAN

X = [[1, 2], [1.5, 1.8], [5, 8], [8, 8],
     [1, 0.6], [9, 11]]

dbscan = DBSCAN(eps=0.5, min_samples=2)
labels = dbscan.fit_predict(X)

# 输出: [ 0  0 -1  1 -1  1]
# 0和1是两个簇，-1是噪声点
```

---

## Python 代码：层次聚类

```python
from sklearn.cluster import AgglomerativeClustering
from scipy.cluster.hierarchy import dendrogram, linkage

X = [[1, 2], [1.5, 1.8], [5, 8], [8, 8]]

clustering = AgglomerativeClustering(n_clusters=2)
labels = clustering.fit_predict(X)

# 绘制树状图
Z = linkage(X, 'ward')
dendrogram(Z)
```

---

## 总结

| 算法 | 核心思想 | 最适合场景 |
|------|----------|------------|
| **DBSCAN** | 基于密度找簇 | 不规则形状、有噪声、不知道簇数 |
| **层次聚类** | 逐步合并/分裂 | 需要层次结构、可视化、小数据集 |

**没有最好的算法，只有最适合的算法**
