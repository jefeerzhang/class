# FP-Growth 与 Apriori 算法对比分析报告

> 基于银行客户金融产品持有数据的关联规则挖掘实验

---

## 1. 实验概述

### 1.1 数据概况

| 指标 | 数值 |
|:----:|------|
| 客户数量 | 100 人 |
| 产品种类 | 6 种 |
| 最小支持度 | 0.25 (25%) |
| 最小置信度 | 0.6 (60%) |

**产品列表**：储蓄账户、基金、保险、信用卡、贷款、理财产品

### 1.2 算法参数

| 算法 | 扫描次数 | 候选集 | 设计思想 |
|------|---------|--------|----------|
| **Apriori** | 多次 | 大量候选项集 | 先验原理剪枝 |
| **FP-Growth** | 2次 | 无候选项集 | FP-Tree压缩 |

## 2. 产品渗透率分析

![产品渗透率](product_penetration.png)

**业务洞察**：储蓄账户是推荐的"钩子产品"——几乎所有客户都有，可作为交叉销售的起点。

## 5. 算法性能对比

### 5.1 执行时间

| 算法 | 耗时 | 数据扫描次数 |
|------|------|-------------|
| **FP-Growth** | 0.0010 秒 | 2次 |
| **Apriori** | 0.0012 秒 | 多次 |

**加速比**：FP-Growth 比 Apriori 快 **1.26 倍**

### 5.2 结果一致性验证

两种算法挖掘出的频繁项集**完全一致**，验证了算法的正确性。

### 5.3 性能差异分析

| 数据规模 | 预期加速比 | 原因 |
|----------|-----------|------|
| 100条 | ~1.2x | 数据量小，差异不明显 |
| 1,000条 | ~2-3x | 候选集开始膨胀 |
| 10,000条 | ~3-5x | 候选项集爆炸 |
| 100,000+条 | ~10x+ | FP-Tree压缩效果显著 |

**注**：当前数据量较小（100条），两种算法差异不明显。大规模数据时，FP-Growth 的优势会显著放大。

## 6. 算法优缺点对比

| 维度 | FP-Growth | Apriori |
|------|-----------|---------|
| **扫描次数** | 2次 | 多次（与项集长度相关） |
| **候选项集** | 无 | 大量（指数级增长） |
| **内存占用** | 中（FP-Tree） | 高（候选项集） |
| **执行速度** | 快 | 慢 |
| **实现复杂度** | 中等 | 简单 |
| **适合场景** | 大规模数据 | 学习、小规模数据 |
| **可解释性** | 相同 | 相同 |

## 8. 结论

### 8.1 算法选择建议

```
数据规模 < 1,000条：
  → 两者皆可，Apriori 更易理解

数据规模 1,000 ~ 10,000条：
  → 推荐 FP-Growth（性能优势明显）

数据规模 > 10,000条：
  → 必须使用 FP-Growth（Apriori 可能内存溢出）
```

### 8.2 核心发现

1. **储蓄账户**是核心产品（91%渗透率），是所有关联分析的"锚点"
2. **基金+保险**组合是最有价值的产品对（lift=1.31）
3. **信用卡+保险 → 基金**规则置信度最高（91.4%），转化效果最好
4. **FP-Growth**在保持相同结果的前提下，性能优于Apriori

### 8.3 下一步建议

1. **扩大数据规模**：收集真实客户数据，验证规则泛化能力
2. **加入时间维度**：分析客户购买顺序，发现时序关联规则
3. **引入客户属性**：年龄、收入、地区等多维分析
4. **A/B测试验证**：小范围测试规则效果后再全量推广
